**提示词：**

请开发一个完整的家庭记账程序，包含前端、后端和 MySQL 数据库。具体要求如下：

1. **技术栈**  
   - 前端：HTML5 + CSS3 + 原生 JavaScript（或 Vue/React 可选，但保持轻量）。  
   - 后端：Python + Flask（提供 RESTful API）。  
   - 数据库：MySQL（存储交易记录）。  

2. **核心功能**  
   - 记账记录的增删改查：支持添加、编辑、删除每一笔收入或支出。  
   - 每条记录包含：日期、描述、分类（如餐饮、购物、交通、工资等）、金额、类型（收入/支出）。  
   - 统计展示：页面顶部显示总余额、总收入、总支出。  
   - 图表可视化：使用 Chart.js 展示支出分类的饼图（仅统计支出）。  
   - 交易流水表格：展示所有记录，支持按时间倒序排列，并带有编辑/删除按钮。  

3. **UI/UX 风格**  
   - 暗黑系主题：背景使用深色渐变（如深灰到黑），卡片采用半透明毛玻璃效果。  
   - 科技感元素：霓虹蓝（#0ff）作为强调色，边框发光、微阴影，字体使用现代无衬线体。  
   - 响应式布局：在大屏幕和手机上都美观可用。  

4. **数据库设计**  
   - 表名：`transactions`  
   - 字段：`id`（自增主键）、`date`（DATE）、`description`（VARCHAR）、`category`（VARCHAR）、`amount`（DECIMAL）、`type`（ENUM('income','expense')）、`created_at`（TIMESTAMP）。  

5. **API 端点设计**  
   - `GET /api/transactions`：获取所有记录。  
   - `POST /api/transactions`：新增记录。  
   - `PUT /api/transactions/<id>`：更新记录。  
   - `DELETE /api/transactions/<id>`：删除记录。  

6. **项目结构**  
   - 提供一个单一 Python 文件（如 `app.py`）作为后端，包含数据库初始化和路由。  
   - 前端静态文件（HTML/CSS/JS）放在 `templates/index.html` 中（或内联样式+脚本）。  
   - 提供 MySQL 连接配置的说明（如使用环境变量或注释）。  

7. **交付要求**  
   - 输出完整可运行的代码，前端页面美观且符合“暗黑科技”风格。  
   - 后端代码包含必要的错误处理和数据库连接池（可选）。  
   - 附带简要运行说明（如何创建数据库、安装依赖、启动服务）。  

请生成上述家庭记账程序的完整代码。
